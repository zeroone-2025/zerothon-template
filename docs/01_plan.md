## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | (프로젝트 이름) |
| 한줄 소개 | (한 문장으로 설명) |
| Pain Point | (해결하려는 문제) |

> **AI 프롬프트** -- 아이디어 구체화하기
>
> ```
> 우리 팀은 2시간 해커톤에서 Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui로 웹 서비스를 만들려고 해.
> 관심 분야: {{관심 분야 (예: 생산성, 교육, 건강, 소셜 등)}}
> 팀 구성: {{팀원 수와 역할}}
>
> 2시간 안에 MVP를 만들 수 있고, 데모 임팩트가 큰 아이디어 3개를 제안해줘.
> 각각 프로젝트명, 한줄 소개, Pain Point, 핵심 기능 3개를 정리해줘.
> ```
>
> **TIP**: 3개 아이디어 중 가장 데모하기 좋은 것을 선택하세요. 기술적으로 어려운 것보다 시각적으로 확인 가능한 것이 발표에 유리합니다.

## MVP 범위

> 2시간 내에 구현 가능한 핵심 기능만 선정하세요.

- [ ] 핵심 기능 1: (설명)
- [ ] 핵심 기능 2: (설명)
- [ ] 핵심 기능 3: (설명)
- [ ] (추가 기능은 우선순위를 매겨 아래에)

> **AI 프롬프트** -- 2시간 MVP 기능 선별
>
> ```
> 서비스: {{한줄 소개}}
> 해결하는 문제: {{Pain Point}}
> 타겟 사용자: {{주요 타겟}}
> 기술 스택: Next.js 15 (App Router), TypeScript, Tailwind CSS, shadcn/ui, Zustand, React Query
>
> 2시간 해커톤에서 구현할 MVP 기능을 선별해줘.
> Must-have(2-3개), Nice-to-have, Out of scope로 나누고
> 각 기능마다 예상 구현 시간과 난이도(상/중/하)를 표시해줘.
> ```
>
> **TIP**: Must-have 기능이 3개를 넘으면 위험 신호입니다. 과감하게 줄이세요.

## 타임라인

| 시간 | 활동 | 담당 |
|------|------|------|
| 0:00 - 0:15 | 아이디어 확정 & 역할 분담 | 전체 |
| 0:15 - 0:30 | 기획 문서 작성 & 화면 설계 | 기획 |
| **0:30** | **체크포인트** — 기획자가 MVP 범위와 화면 설계 최종 확인 | **기획자** |
| 0:30 - 1:30 | 핵심 기능 개발 | 개발 |
| **1:30** | **체크포인트** — 기획자가 개발/마케팅 결과물이 기획 방향과 맞는지 확인 | **기획자** |
| 1:30 - 1:45 | 통합 테스트 & 버그 수정 | 전체 |
| 1:45 - 2:00 | 발표 준비 | 전체 |

> **TIP**: MVP 기능마다 시간을 배분하세요. 예를 들어 기능이 3개라면 개발 60분을 20분씩 나누고, 마지막 10분은 버퍼로 남겨두세요.
