# 📚 문서 워크플로우 가이드

> 제로톤 2시간 해커톤에서 AI를 활용해 기획부터 발표까지 빠르게 완성하는 가이드입니다.

---

## 문서 흐름

```
01_plan.md (기획)  →  02_dev.md (개발)  →  03_marketing.md (마케팅)  →  04_ppt.md (발표)
  │                  │                    │                       │
  │  아이디어 정리    │  설계 & 구현       │  타겟 & 메시지        │  스크립트 종합
  │  MVP 범위 확정   │  API/컴포넌트      │  카피 & 콘텐츠        │  데모 시나리오
  └──────────────────┴────────────────────┴───────────────────────┘
```

각 문서의 출력이 다음 문서의 입력이 됩니다. 순서대로 작성하면 자연스럽게 연결됩니다.

> **기획자는 팀 리더입니다.** 아이디어 선정, MVP 범위, 타임라인을 최종 결정하고,
> 개발/마케팅 결과물이 기획 방향과 맞는지 확인합니다.

---

## 역할별 시작 가이드

| 역할 | 먼저 열 문서 | 함께 볼 문서 | 설명 |
|------|-------------|-------------|------|
| **기획자** | `01_plan.md` | - | **팀 리더.** 아이디어 확정, MVP 범위 결정, 전체 방향 조율 |
| **개발자** | `02_dev.md` | `01_plan.md` | 기획 기반으로 기술 설계 & 구현 |
| **마케터** | `03_marketing.md` | `01_plan.md`, `02_dev.md` | 타겟 분석, 메시지, 콘텐츠 기획 |
| **발표자** | `04_ppt.md` | 전체 문서 | 3개 문서를 종합하여 발표 스크립트 작성 |

> 소규모 팀이라면 한 사람이 여러 역할을 겸할 수 있습니다. 순서만 지키세요!

## Claude Code에서 문서 활용하기

각 문서에는 **AI 프롬프트 블록**이 포함되어 있습니다. Claude Code의 `@` 파일 참조를 활용하면 문서 간 내용을 자동으로 연결할 수 있습니다.

**사용 방법:**
1. Claude Code에서 `@docs/01_plan.md` 형식으로 문서를 참조
2. 프롬프트 블록의 내용을 참고하여 자연어로 요청
3. `{{플레이스홀더}}`는 여러분의 내용으로 직접 채워 주세요
4. 결과를 해당 섹션에 정리

**예시:**
```
@docs/01_plan.md 참고해서, 핵심 기능 구현에 필요한 추가 라이브러리를 추천해줘.
번들 크기와 러닝커브 고려해서 설치 명령어와 선택 이유 포함해줘.
```

**도구:** Claude Code (터미널에서 `claude` 명령으로 실행)

## 2시간 타임라인 가이드

| 시간 | 활동 | 관련 문서 |
|------|------|----------|
| **0:00 - 0:15** (15분) | 아이디어 확정, 역할 분담 | `01_plan.md` |
| **0:15 - 0:30** (15분) | 기획 문서 작성, 화면 설계 | `01_plan.md`, `02_dev.md` 상단 |
| **0:30 - 1:30** (60분) | 핵심 기능 개발 | `02_dev.md` |
| **1:00 - 1:30** (30분) | 마케팅 문서 작성 (개발과 병행) | `03_marketing.md` |
| **1:30 - 1:45** (15분) | 통합 테스트, 버그 수정 | `02_dev.md` |
| **1:45 - 2:00** (15분) | 발표 스크립트 작성, 리허설 | `04_ppt.md` |

> 문서 작성 자체에 너무 많은 시간을 쓰지 마세요. AI 프롬프트를 활용하면 각 문서를 **5-10분** 안에 채울 수 있습니다.
