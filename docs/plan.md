# 📋 기획 문서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 제로톤 팀 랜덤 매칭 |
| 한줄 소개 | 참가자 이름을 입력하면 기획·마케팅·개발 역할을 자동으로 랜덤 배정해주는 도구 |
| Pain Point | 제로톤 시작 전 역할 분담을 수동으로 정하느라 시간을 낭비함 |

## 타겟 사용자

- **주요 타겟**: 제로톤 진행자(운영팀) 및 참가자
- **사용자 특성**: 2시간 해커톤에서 빠르게 팀을 구성해야 하는 상황
- **시장 규모**: 제로톤 참가자 전원 (매주 수요일)

## MVP 범위

> 1시간 내에 구현 가능한 핵심 기능만 선정 — API 없이 클라이언트 상태만 사용

- [ ] **MVP 1: 팀원 이름 입력 및 목록 관리**
  - 이름 입력 필드 + 추가 버튼
  - 추가된 팀원 목록 표시 (삭제 가능)
  - 최소 3명 이상 입력 시 배정 버튼 활성화

- [ ] **MVP 2: 역할 랜덤 배정**
  - [기획 / 마케팅 / 개발] 3가지 역할로 참가자를 균등하게 셔플 배정
  - 역할 수가 인원보다 많으면 순환 배정 (예: 4명 → 기획 2, 마케팅 1, 개발 1)
  - 배정 로직은 Fisher-Yates 셔플 알고리즘 사용

- [ ] **MVP 3: 결과 카드 표시 + 다시 뽑기**
  - 역할별 카드(기획/마케팅/개발)에 배정된 이름 목록 표시
  - "다시 뽑기" 버튼으로 즉시 재배정
  - 카드별 색상 구분 (기획: 파랑, 마케팅: 초록, 개발: 보라)

## 화면 구성 (1페이지)

```
[제로톤 팀 매칭]

이름 입력: [        ] [추가]
참가자 목록: 김철수 ✕  이영희 ✕  박민수 ✕  ...

[역할 배정하기] 버튼

--- 결과 ---
🎯 기획    💻 개발    📢 마케팅
김철수      이영희      박민수
...

[다시 뽑기]
```

## 기술 구현 계획

| 파일 | 역할 |
|------|------|
| `src/app/page.tsx` | 메인 페이지 (입력 + 결과) |
| `src/stores/use-team-store.ts` | 팀원 목록 & 배정 결과 Zustand 스토어 |
| `src/components/TeamInput.tsx` | 이름 입력 컴포넌트 |
| `src/components/ResultCards.tsx` | 역할별 결과 카드 컴포넌트 |

## 타임라인

| 시간 | 활동 | 담당 |
|------|------|------|
| 0:00 - 0:10 | 기획 확정 & plan.md 작성 | 기획 |
| 0:10 - 0:20 | Zustand 스토어 + 셔플 로직 구현 | 개발 |
| 0:20 - 0:40 | 입력 UI + 결과 카드 컴포넌트 구현 | 개발 |
| 0:40 - 0:50 | 스타일링 (Tailwind) & 다시뽑기 연결 | 개발 |
| 0:50 - 1:00 | 테스트 & 버그 수정, 발표 준비 | 전체 |

## 비즈니스 모델

- **수익 모델**: 내부 운영 도구 (수익화 불필요)
- **차별점**: 제로톤 3개 역할에 특화된 즉시 사용 가능한 배정 도구
- **확장 가능성**: 팀 수 설정, 배정 결과 공유 링크, Slack 연동
